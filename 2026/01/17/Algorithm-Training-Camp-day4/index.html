24. 两两交换链表中的节点

1. 我的思路

  • 首先观察，我们的链表如下，那其实我们要做的很简单，就是想办法让Node0与Node1交换位置 
    	graph LR
      Node0-->Node1-->...
  • 但是直接交换的话，对于头部的管理就会有点令人难受。这个时候，添加一个虚拟头节点_dummy_head就会方便很多，因为此时 原链表的头节点也可以作为普通节点参与后续的运算 
    	graph LR
      _dummy_head-->Node0
      Node0-->Node1-->...
  • 此时，我们进入正式交换，其实逻辑很简单：
    1. 将虚拟头节点从原来指向Node0改为指向Node1
    2. 将Node0从原来指向Node1改为指向...
    3. 将Node1从原来指向...改为指向Node0 
      	graph LR
   _dummy_head~~~Node0~~~Node1~~~...
   _dummy_head-->Node1-->Node0-->...
  • 而为了推广到一般情况，我用prev、curr、next替代了具体的节点，其中：
    • prev是不会改变具体连接的，起锚定作用的（类似于前文的_dummy_head）
    • curr与next是会在流程中产生交换的节点（类似于前文的Node0与Node1）
      	graph LR
    etc_1[...]-->prev-->curr-->next-->etc_2[...]
  • 而此时具体的交换就如下了：
    1. 将prev从原来指向curr改为指向next
    2. 将curr从原来指向next改为指向...
    3. 将next从原来指向...改为指向curr 
      	graph LR
   etc_1[...]-->prev~~~curr~~~next~~~etc_2[...]
   prev-->next-->curr-->etc_2[...]
  • 接着就是移动节点和判断停止了，停止条件很简单就是检查 再往后的两个节点，是否存在。其中但凡有一个不存在，那就结束了
  • 不过实操中，你应该注意到：
    1. 交换后，原来的curr，才是现在的3个节点（prev、curr、next）中最靠后的节点！

19.删除链表的倒数第N个节点

1. 我的思路

这道题如果要满足一次循环结局的话，其实思路也挺固定的。
就像数组快慢指针，只要两个指针之间恒定一个距离，保持这个距离移动，那么当快指针移动到尾巴的时候，慢指针自然而然就指向了待删除的元素

不过实际移动中，我有以下细节上的修改：

  • 我指向的是待删除元素的前一个元素：
    • 这样可以确保删除的时候，与前面链表的结合正常（因为这是单向链表）
  • 添加了虚拟头节点
    • 这样我可以将原来的头节点也当作一般节点处理

时间复杂度: O(n)
空间复杂度: O(1)

160.链表相交

1. 我的思路

*这里我的思路就简写了，参考了卡哥的思路。但是我觉得这个思路其实比较普通。

我会着重笔墨描写其它思路中的双指针思路，这个思路我评价为惊艳！

卡哥的思路其实很简单：

  • 由于如果有重叠部分，那重叠部分的长度对于两个链表来说，长度是一致的
  • 遍历链表A和链表B， 计算长度差
  • 移动二次遍历时二者的起点，使得剩余链表的长度一致
  • 开始二次比较，逐节点比较，直至结束

2. 其它优秀思路

1. 哈希

这个方案其实就比较直接了，遍历A，将其每一个地址放入一个哈希集合。
再遍历B，依次检查是不是存在于集合中
若存在，则返回，否则最后返回NULL。

其实说白了，就是遍历，不过使用哈希来加快速度

2. 双指针

这个思路我觉得是最最棒的思路！按照这个思路写出来的代码简洁，且同样满足时间复杂度 O(n)，空间复杂度O(1)!

其具体行为如下：

  • 这是我们的初始链表 
    	graph LR
      node_A0-->node_A1-->node_A2-->node_C0-->node_C1-->NULL
      node_B0-->node_B1-->node_C0
  • 双链表的行为是：
    • 两个指针分别指向两个链表的头节点，并一起移动
    • 当其中一个指针移动到NULL，且另一个指针并非NULL时，NULL的指针移动到另一链表的头部。
    • 如果某一时刻，两个指针指向相同节点或都为NULL，返回该节点
    • 重复，直到结束
  • 由于一次循环会走遍两个链表，所以长度最大为 m + n （其中m、n为两个链表的长度）
  • 且由于总长度一样，一定会同时停止
  • 此外，观察他们最终的几个路径，一定都是相同的路径 C
  • 所以，双指针的方式最终一定会走到相交的节点（如果有），如图所示
    • pA 轨迹
      	graph LR
    direction LR
    subgraph List A
        direction LR
        node_A0_pA[node_A0]-->node_A1_pA[node_A1]-->node_A2_pA[node_A2]-->node_C0_time1_pA[node_C0]-->node_C1_time1_pA[node_C1]-->NULL_time0_pA[NULL]
    end
    subgraph List B
        direction LR
        NULL_time0_pA[NULL]-->node_B0_pA[node_B0]-->node_B1_pA[node_B1]-->node_C0_time2_pA[node_C0]-->node_C1_time2_pA[node_C1]-->NULL_time1_pA[NULL]
    end
    • pB 轨迹
      	graph LR 
    direction LR
    subgraph List B
        direction LR
        node_B0_pB[node_B0]-->node_B1_pB[node_B1]-->node_C0_time1_pB[node_C0]-->node_C1_time1_pB[node_C1]-->NULL_time0_pB[NULL]
    end
    subgraph List A
        direction LR
        NULL_time0_pB[NULL]-->node_A0_pB[node_A0]-->node_A1_pB[node_A1]-->node_A2_pB[node_A2]-->node_C0_time2_pB[node_C0]-->node_C1_time2_pB[node_C1]-->NULL_time1_pB[NULL]
    end
  • 而如果没有相交节点，那么你可以视作不会经过上图中任何C相关节点，很显然，最后会指向NULL，那么就返回NULL，表示没有相交节点。
  • 如果长度相同，那么很显然，会在前半次循环就结束

这是一个迅速高效的方案！
时间复杂度 O(m+n)
空间复杂度 O(1)

24. 两两交换链表中的节点

这题我的思路卡在双指针后，如何判断具体节点
所以此处我就介绍一下两种不同的方法

1. 双指针

具体思路如下

  • 我们设置两个指针，一快一慢，记作fast，和slow。
    • 其中fast每次移动两个节点，slow每次移动一个节点
    • 显然，如果链表无环，最终肯定有一个会到达null，此时返回null即可
    • 如果有环，肯定会相遇
      • 且由于移动速度差一倍，显然无论初始移动距离差多少，在slow于环内转动一圈结束前，fast肯定会与slow相遇
  • 接着，我们就要考虑，如何根据当前位置，判断环的入口了

    这就是当时困扰我的

    • 我们不妨进行如下假设：
      • 假设在环入口前的距离为 x
      • 假设相遇位置距离环起点距离为 y
      • 假设相遇位置距离再次到达起点距离为 z
      • 假设相遇时fast已经循环了 n 圈（其中，n肯定>=1） 
        		graph LR
      起点-->|x|入口-->|y|相遇点-->|z|入口
    • 我们可以确认，当相遇时，路程刚好为两倍关系，所以有：
      • 2 * slow路径长度 = fast路径长度
      • 2 * (x + y) = x + n * (y + z) + y
    • 由于我们要求入口，所以尝试将 x 放到等式左侧，化简可以得到
      • x = n * (y + z) - y
      • x = (n - 1)(y + z) + z
    • 这个时候，你就能发现，两个指针，一个再次从起点出发，另一个从相遇点触发
      • 记从起点出发的为 index1， 相遇点出发的为 index2
      • 两者移动速度均为 1 个节点
      • 当 index1 移动到入口时，移动了 x 个节点，你应该注意到：
        • 这段路程中，无论 index2 移动了多少给节点转动了多少圈，两者始终都没有相遇过
      • index2 无论移动多少个节点，由于有 (n - 1) 次环形移动，所以相对于相遇点的移动路径始终是 z
    • 这说明，这两个 index1 和 index2 的相遇位置，就是节点的入口
