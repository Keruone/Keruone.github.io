203. 链表元素移除

题目链接：https://leetcode.cn/problems/remove-linked-list-elements/
文章讲解：https://programmercarl.com/0203.%E7%A7%BB%E9%99%A4%E9%93%BE%E8%A1%A8%E5%85%83%E7%B4%A0.html#%E6%80%9D%E8%B7%AF

1. 初始思路

这道题就是最基础的链表，我的思路如下：

  • 创建一个自己的头节点，直接连接到输入的head
    • 这样就可以让整个输入链表都作为非头节点处理了
    • 且我的头节点必然存在，非NULL
  • 然后就是通用的对于非头节点的处理思路：
    1. 首先，创建一个一直移动检查的节点指针 index，指向我的头节点

      此时我的 index 必然不是 NULL

    2. 接着开始检查 index 的 next 指向的节点
      1. 如果为 NULL ，那么没必要检查了，直接退出循环
      2. 否则检查 next对应的val，即index->next->val:
        1. 如果该 val 是要删除的数值，那么就该跳过这个next节点

          那么就是 index->next = index->next->next
          由于 index->next 存在，所以index->next的next肯定有指向，哪怕是NULL。

        2. 否则就可以正常移动index到下一个节点

          因为前文判断过 index->next肯定是存在的
          所以放心移动，不会出事的

    3. 重复步骤2，直到退出
    4. 此时，我的头节点指向的下一个节点，就是我应该返回的节点

      不过记得 delete 不需要使用的节点

2. 卡哥思路

本质其实差不多，实现上的差别就是：

  • 卡哥是先循环，特殊处理头节点
  • 但我是增加一个节点，指向头节点，然后统一处理全部的节点

206.反转链表

题目连接：https://leetcode.cn/problems/reverse-linked-list/description/
文章讲解：https://programmercarl.com/0206.%E7%BF%BB%E8%BD%AC%E9%93%BE%E8%A1%A8.html

1. 我的思路

其实我的想法很简单，即:

  • 一直移动head，同时沿途反转head->next指向，直到结束
    • 此时head指向的就是新链表的头节点了

具体来说，就是

  • 记录前一个节点prev和后一个节点next
    • 其中prev初始值为nullptr，对应反转后最后指向空指针
    • 其中next一直来自head->next，直到结束

      因为连接转向后，会寻不见next，所以要提前记录

  • 重复执行：
    • 更新存储当前next： next = head->next
    • 更改连接朝向：head->next = prev
    • 此时，完成了一次反转，接下来，我们应该一并移动 prev、head、next

时间复杂度O(n)
额外空间复杂度O(1)

2. 卡哥思路

这道题其实我的思路和卡哥的本质上也是相同的，所以就不多赘述了。
不过卡哥返回的是prev，这样就可以将整个操作放进单个循环里，会更加直观些。
