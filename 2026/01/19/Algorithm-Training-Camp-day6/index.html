349. 两个数组的交集

其实这道题思路还是很直接的：

  • 对于两个数组，先遍历第一个，存入哈希集合中，
  • 再遍历第二个数组，且每次与第一个哈希集合检查，如果有重复的，则保存到新集合中
    • 由于这里不要重复的输入，所以使用集合来保存最终内容，因为哈希集合不会有重复数据

此外，有些以前没见过的 cpp 特性，在此做简单介绍：

  • for (declaration : expression)（参考这里）:
    • declaration： 表示定义的变量，类型为要遍历序列中存储元素的类型，或者 auto(c++11)
    • expression：表示要遍历的序列，常见的可以为事先定义好的普通数组或者容器，还可以是用 {} 大括号初始化的序列。
    • 不过这里的 declaration 是 值拷贝，比较不同。
    • 如果想要直接更改原序列的内容，可以使用 引用（&），例如for (char &ch : string)
      • 可以在循环体内修改 ch 的值，且没有拷贝开销。
    • 如若没有修改，也可以使用**常引用(const auto &ch)**，如for (const char &ch : string)
      • 只读访问。既避免了拷贝（性能好），又保证了数据不会被意外修改

1. 两数之和

枚举思路略过。
这道题可以使用哈希一遍过，最终复杂度 O(N)

  • 使用哈希的方式，构建一个哈希集合
  • 暂记当前数字为 num，那么对于这个数字来说的期望数字就是 expect = target - num
  • 如果我在集合中找不到 expect， 那我就把当前的 num 和对应的索引 i 加入到集合中，因为如果目标是 O(N) 的话，那这个元素就不会再次经过了，也正因此，需要键值对
  • 如果找到了，则返回

考虑到：

  • 要使用 键值对，所以建议使用 map
  • 同时，考虑到不需要顺序，所以可以 unordered_map
  • 注意，std::xxx_map 仅有两个公开成员
    • ->first：指向 键（Key）
    • ->second：指向 值（Value）

迭代器解释

由于这次学习中，发现很多与迭代器相关的内容，有点云里雾里，所以此处写出来介绍一下

豆包简介：

  • 在 C++ 中，迭代器（Iterator） 是一个非常核心的概念。
  • 用一句话概括：迭代器是一个 “对象”，它的行为像一个指针，用来指向容器（如数组、链表、哈希表等）中的元素，并提供了访问这些元素的方法。
  • 它的设计目的是为了统一不同容器的访问方式。

迭代器的基本语法（像指针一样用）

迭代器的使用方式和指针非常像，主要重载了以下运算符：

  • *it (解引用)：
    获取迭代器指向的元素的值。
  • cout << *it;
    it-> (箭头)：
    如果元素是一个对象（如 pair），访问该对象的成员。
    • cout << it->first;
  • ++it / it++ (自增)：
    移动迭代器，使其指向下一个元素。
  • --it / it-- (自减)：
    移动迭代器，使其指向前一个元素（注意：不是所有迭代器都支持）。
  • it->end()：
    这是一个尾后迭代器。它不指向任何有效元素，而是指向容器最后一个元素后面的位置。它是循环结束的标志。

迭代器的分类（能力不同）

并不是所有迭代器都一样强。根据功能强弱，C++ 标准定义了 5 种主要类型：

  1. 输入 / 输出迭代器 (Input/Output Iterator)：
    • 只能读 / 写一次，只能向前移动。
    • 比如：std::istream_iterator。
  2. 前向迭代器 (Forward Iterator)：
    • 可以多次读写，只能向前移动。
    • 比如：std::forward_list（单向链表）。
  3. 双向迭代器 (Bidirectional Iterator)：
    • 可以向前，也可以向后移动。
    • 比如：std::list（双向链表）、std::map、std::set。
  4. 随机访问迭代器 (Random Access Iterator)：
    • 功能最强。支持跳跃访问（it + 5）、比较大小（it1 < it2）。
    • 比如：std::vector、std::deque、普通数组指针。

这就是为什么 map 不能用 it + 1，而 vector 可以。 因为 map 的迭代器只是双向的，不支持随机跳跃。
